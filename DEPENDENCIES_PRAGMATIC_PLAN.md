## 🎯 务实方案：部分拆分

### 方案：只拆分最大的 3-4 个模块

```
app/api/
├── dependencies.py (~1200 行) - 保留核心依赖和中小型函数
├── utils/
│   ├── document_helpers.py (~450 行) - 文档相关（14 个函数）
│   ├── query_helpers.py (~350 行) - 查询相关（已创建）

```

### 收益

- dependencies.py: 2000 行 → ~1200 行 (**-40%**)

- 实施时间: **30-45 分钟**（而不是 2-3 小时）
- 风险: **低**（只动 3 个模块）
