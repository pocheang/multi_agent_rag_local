# 项目工作记录

## 项目概述
本地化多智能体 RAG 系统（FastAPI + LangGraph + React），实现文档索引、混合检索、图谱检索、流式问答、权限管理与运维治理。通过分层执行策略优化查询响应延迟，支持企业私有化部署。

## 项目信息
- **项目名称**：Multi-Agent Local RAG（企业知识问答平台）
- **开发周期**：2026.02 - 至今
- **当前版本**：v0.2.4（2026-04-26）
- **历史版本**：v0.1.0、v0.2、v0.2.1、v0.2.2、v0.2.2.1

## 技术架构
**后端技术栈**
- 框架：Python 3.11、FastAPI、Pydantic、Uvicorn
- 智能体编排：LangGraph、LangChain
- 检索引擎：ChromaDB（向量）、BM25（关键词）、BAAI/bge-reranker-v2-m3（重排）
- 图数据库：Neo4j
- LLM 接入：OpenAI API、Anthropic Claude、Ollama
- 缓存与队列：Redis、内存缓存
- 工具库：Sentence-Transformers、DDGS、HTTPX、Tenacity

**前端技术栈**
- React + TypeScript + Vite

**工程化**
- 测试：Pytest（单元测试、集成测试、并发测试）
- 容器化：Docker Compose（Neo4j）
- 配置管理：环境变量（`.env`）

## 技术实现

### 1. 多智能体工作流
- 基于 LangGraph 实现 5 个智能体协作：Router Agent（路由）、Vector RAG Agent（向量检索）、Graph RAG Agent（图谱检索）、Web Research Agent（网络搜索）、Synthesis Agent（答案合成）
- 实现查询分类与策略路由（pdf_text、cybersecurity、general 等类型）
- 状态流转通过 TypedDict 定义的 GraphState 传递

### 2. 混合检索系统
- 向量检索：ChromaDB，支持相似度阈值配置
- 关键词检索：BM25，基于语料库存储
- 融合策略：RRF（Reciprocal Rank Fusion）
- 重排序：BAAI/bge-reranker-v2-m3
- 分块策略：Parent-Child Chunking（小块检索，大块上下文）
- 缓存机制：TTL 缓存（内存/Redis），支持重复查询加速

### 3. 图谱检索增强
- Neo4j 图数据库集成
- 实体关系提取与存储
- 结构化信息补充答案生成

### 4. 文档索引流程
- 支持格式：PDF、图片（OCR）、文本文档
- 自动增量索引：文件监听，稳定后入库
- 防重机制：避免脏数据与重复写入

### 5. 权限与安全
- 认证：JWT Token
- 授权：RBAC（Role-Based Access Control）
- 数据隔离：用户级数据可见性控制（私有/公共）
- 审计日志：操作记录与过滤查询
- 高权限保护：管理员创建需审批令牌、密码重置、审批令牌重置

### 6. 管理后台
- 运营看板：KPI 展示、趋势分析
- 数据导出：CSV 格式
- 审计日志：操作记录查询与过滤
- 用户管理：角色/状态/分类管理

### 8. 查询延迟优化（v0.2.4，2026-04-26）
**分层执行策略**
- 三层分类：fast（快速）、balanced（平衡）、deep（深度）
- 分类依据：查询复杂度、文档命中概率、查询长度、用户显式请求
- 实现位置：查询分类器（tier_classifier.py，规划中）

**延迟预算管理**
- Fast 层级：retrieval_top_k=5, rerank_top_k=3, max_retrieval_time=800ms, max_synthesis_tokens=300, web_fallback=disabled
- Balanced 层级：retrieval_top_k=10, rerank_top_k=5, max_retrieval_time=2000ms, max_synthesis_tokens=800, web_fallback=conditional
- Deep 层级：retrieval_top_k=20, rerank_top_k=10, max_retrieval_time=5000ms, max_synthesis_tokens=1500, web_fallback=enabled
- 实现位置：延迟预算管理器（budget_policy.py，规划中）

**负载感知降级**
- 系统负载 >80%：自动降一级
- 系统负载 >95%：强制 fast 层级
- 保障高并发场景稳定性

**Web 回退触发条件**
- 本地证据置信度 <0.5
- 查询包含时间关键词（latest、recent、current、2026）
- 用户显式请求网络搜索
- 文档语料库更新时间 >30 天（时间敏感查询）

**UX 遥测**
- 首 token 延迟：P50、P95、P99
- 层级分布：fast/balanced/deep 百分比
- 层级置信度：分类准确性监控
- 引用覆盖率：每层级引用质量
- 重问率：5 分钟内相似查询（embedding 距离 >0.7）
- 预算超时计数：每层级每组件超时统计
- 负载降级计数：降级触发频率

**流式响应优化**
- 渐进式证据交付
- 响应头透出层级元数据：X-Query-Tier、X-Tier-Confidence
- 向后兼容：不请求元数据的客户端不受影响

**延迟目标达成**
- 首 token 延迟：P50 ≤ 2s、P95 ≤ 4s
- 端到端答案延迟：P50 ≤ 12s（balanced 层级）
- 相比优化前延迟降低：40-60%
