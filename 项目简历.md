# 项目经历（简历版）

## 一句话版本（可放简历项目标题下）
独立开发并持续迭代本地化多智能体 RAG 系统（FastAPI + LangGraph + React），实现文档上传索引、混合检索、图谱检索、流式问答、权限隔离与管理员运维看板，具备可部署、可审计、可测试的工程化能力。

## 项目名称
Multi-Agent Local RAG（企业内部知识问答与运维管理平台）

## 项目时间
2026.04 - 至今（v0.1.0 / v0.2 持续迭代）

## 技术栈
- 后端：Python 3.11、FastAPI、Pydantic、Uvicorn
- 智能体与检索：LangGraph、LangChain、ChromaDB、Neo4j、BM25、Reranker
- 模型与工具：OpenAI API / Ollama、Sentence-Transformers、DDGS、Tenacity、HTTPX
- 前端：React + TypeScript + Vite
- 工程化：Pytest、Docker Compose（Neo4j）、环境化配置（`.env`）

## 项目背景
面向本地部署和私有数据场景，构建支持多轮对话、知识检索、权限隔离与管理员运营分析的 RAG 系统，兼顾效果、稳定性与安全合规。

## 核心职责
- 负责整体架构设计与核心模块开发（多智能体编排、检索链路、API、权限与审计）。
- 设计并实现文档处理与索引流程（PDF/图片 OCR、分块、增量入库、自动监听）。
- 打通前后端链路，落地会话管理、流式问答、管理后台、运维指标与导出能力。
- 建立回归测试体系，覆盖安全、权限、检索、管理接口与稳定性场景。

## 关键实现
1. 多智能体工作流与路由
- 基于 LangGraph 设计 Router、Vector RAG、Graph RAG、Web Research、Synthesis 等角色协作流程。
- 引入查询分类（如 `pdf_text` / `cybersecurity` / `general`）提升路由精度与响应策略匹配度。

2. 混合检索与图谱检索
- 融合向量检索（Chroma）与关键词检索（BM25），并通过重排模块优化召回结果质量。
- 接入 Neo4j 图谱检索能力，支持结构化关系信息补充答案。

3. 文档处理与自动增量索引
- 支持 PDF、图片 OCR、文本类文档加载，统一进入索引流水线。
- 实现目录监听服务 `AutoIngestWatcher`，仅在文件稳定后触发入库，避免重复与脏数据写入。

4. 安全治理与后台管理
- 实现 JWT 认证、RBAC 权限模型、用户级数据隔离及可见性控制（私有/公共）。
- 新增管理员运营接口（`/admin/ops/overview`、`/admin/ops/export.csv`）与审计日志过滤能力。
- 支持管理员账号创建审批令牌校验、密码重置、审批令牌重置等高权限操作。

5. 前端与交互
- 构建聊天、登录、管理、架构展示等页面，支持流式响应与管理操作闭环。
- 支持开发态与生产态构建部署，后端统一托管生产静态资源。

## 项目结果
- 完成从“文档导入 -> 检索问答 -> 会话管理 -> 管理审计”的端到端闭环。
- 形成可本地化部署、可权限隔离、可审计追踪的企业知识问答底座。
- 持续发布版本（v0.1.0、v0.2），新增管理员运维看板与安全管理能力。
- 建立较完整测试覆盖：`25` 个测试文件、`112` 个测试用例（按当前仓库统计）。
