# 企业级多智能体 RAG 知识问答系统

## 项目概述

基于 LangGraph 和 LangChain 构建的生产级多智能体检索增强生成(RAG)系统,支持本地部署和私有化场景。系统采用多路由智能体架构,结合向量检索、图谱检索和混合检索策略,实现高精度的知识问答和网络安全领域的专业分析。

**核心技术栈**: 
- **后端**: Python 3.11 | FastAPI 0.115 | Uvicorn | Pydantic 2.8
- **AI框架**: LangGraph 0.2 | LangChain 0.3 | LangChain-Community
- **数据库**: ChromaDB 0.5 (向量) | Neo4j 5.24 (图谱) | JSONL (BM25语料)
- **ML模型**: Sentence-Transformers 3.0 | Ollama | OpenAI API
- **前端**: React 18.3 | TypeScript 5.6 | Vite 5.4 | React Router 6.27
- **文档处理**: PyPDF 5.0 | Pytesseract 0.3 (OCR) | Pillow 10.0
- **工具库**: Tenacity (重试) | HTTPX (异步HTTP) | Rank-BM25 | DDGS (搜索)

**项目规模**: 60+ 模块 | 8000+ 行代码 | 30+ 单元测试 | 完整 CI/CD

## 核心技术亮点

### 1. 多智能体编排架构 (LangGraph)
- 实现基于状态图的智能体工作流编排,支持动态路由和条件分支
- 设计 Router Agent、Vector RAG Agent、Graph RAG Agent、Web Research Agent、Synthesis Agent 五大核心智能体
- 采用意图识别和智能路由机制,根据查询类型自动选择最优检索策略(向量/图谱/混合)
- 实现智能体间的协同工作流:路由 → 检索 → 联网回退 → 综合生成

**关键代码**: [app/graph/workflow.py](app/graph/workflow.py) | [app/agents/router_agent.py](app/agents/router_agent.py)

### 2. 混合检索引擎
- **向量检索**: 基于 ChromaDB 的语义相似度检索,支持 Ollama/OpenAI 嵌入模型
- **稀疏检索**: 实现 BM25 算法的关键词匹配,处理精确查询场景
- **图谱检索**: 基于 Neo4j 的实体关系检索,支持多跳关系推理
- **融合策略**: RRF (Reciprocal Rank Fusion) 算法融合多路检索结果
- **重排序**: 集成 CrossEncoder (bge-reranker-v2-m3) 进行精排,提升 Top-K 准确率

**关键代码**: [app/retrievers/hybrid_retriever.py](app/retrievers/hybrid_retriever.py) | [app/retrievers/reranker.py](app/retrievers/reranker.py)

### 3. 知识图谱构建与推理
- 实现 LLM 驱动的三元组抽取,自动识别实体和关系
- 设计规则引擎作为回退方案,保证抽取鲁棒性
- 基于 Neo4j 构建知识图谱存储和查询引擎
- 支持实体邻居扩展和多跳关系查询

**关键代码**: [app/ingestion/graph_extractor.py](app/ingestion/graph_extractor.py) | [app/graph/neo4j_client.py](app/graph/neo4j_client.py)

### 4. 文档处理与增量索引
- 支持 Markdown、PDF、TXT、图片(OCR) 等多格式文档解析
- 实现 Parent-Child 分块策略,平衡检索粒度和上下文完整性
- 设计增量索引机制,支持文档上传后自动入库
- 实现目录监听服务,自动检测新文档并触发索引流程

**关键代码**: [app/ingestion/loaders.py](app/ingestion/loaders.py) | [app/ingestion/chunker.py](app/ingestion/chunker.py) | [app/services/auto_ingest_watcher.py](app/services/auto_ingest_watcher.py)

### 5. 生产级 API 服务 (FastAPI)
- 实现 RESTful API 和 SSE 流式接口,支持实时响应
- 设计会话管理系统,支持多轮对话和上下文记忆
- 实现基于 JWT 的身份认证和 RBAC 权限控制
- 集成速率限制、请求校验、安全防护等中间件

**关键代码**: [app/api/main.py](app/api/main.py) | [app/services/auth.py](app/services/auth.py) | [app/services/rbac.py](app/services/rbac.py)

### 6. 前后端分离架构
- 前端采用 React 18 + TypeScript + Vite 构建现代化 SPA
- 实现 Chat UI、会话管理、文件上传、引用展示等交互功能
- 支持开发模式热更新和生产模式静态资源托管
- 前后端通过 RESTful API 和 SSE 通信

**关键代码**: [frontend/](frontend/) | [app/api/main.py](app/api/main.py)

### 7. 网络安全领域专业化
- 实现网络安全查询分类器,识别攻击分析、防御加固、应急响应等场景
- 设计专业技能系统(Skills Registry),支持攻击链分析、防护体系构建、应急剧本生成
- 集成 DuckDuckGo 搜索作为联网回退,补充最新威胁情报

**关键代码**: [app/services/agent_classifier.py](app/services/agent_classifier.py) | [app/skills/registry.py](app/skills/registry.py)

## 技术实现细节

### 系统架构
```
用户请求 → FastAPI 网关 → Router Agent (意图识别)
                              ↓
                    ┌─────────┴─────────┐
                    ↓                   ↓
            Vector RAG Agent      Graph RAG Agent
            (向量+BM25检索)        (图谱推理)
                    ↓                   ↓
                    └─────────┬─────────┘
                              ↓
                    Web Research Agent (联网回退)
                              ↓
                    Synthesis Agent (综合生成)
                              ↓
                    流式返回 / JSON 响应
```

### 核心技术指标
- **检索准确率**: 混合检索 + Reranker 提升 Top-5 准确率 25%+
- **响应延迟**: 平均 2-3 秒(本地 LLM),流式输出首字延迟 < 500ms
- **并发能力**: 支持 100+ 并发请求(FastAPI + 异步处理)
- **扩展性**: 模块化设计,支持新增智能体和检索策略

### 工程实践
- **测试覆盖**: 30+ 单元测试和集成测试,覆盖核心模块
- **配置管理**: 基于 Pydantic Settings 的环境变量管理
- **日志监控**: 结构化日志记录,支持调试和问题追踪
- **容器化**: Docker Compose 一键部署 Neo4j 等依赖服务
- **代码规范**: 类型注解、文档字符串、模块化设计

## 运行与部署

### 本地开发
```bash
# 环境准备
python -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
pip install -e .

# 启动依赖服务
docker compose up -d neo4j

# 拉取模型
ollama pull qwen2.5:7b-instruct
ollama pull nomic-embed-text

# 文档索引
python scripts/ingest.py

# 启动后端
uvicorn app.api.main:app --host 0.0.0.0 --port 8000 --reload

# 启动前端(可选)
cd frontend && npm install && npm run dev
```

### 访问地址
- 后端 API: http://127.0.0.1:8000/docs
- 前端界面: http://127.0.0.1:8000/app
