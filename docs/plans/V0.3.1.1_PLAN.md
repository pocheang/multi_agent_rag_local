# v0.3.1.1 版本计划

**版本号**: v0.3.1.1  
**计划日期**: 2026-04-28  
**发布类型**: 补丁版本 (Patch Release)  
**优先级**: P1 - 功能修复

## 概述

v0.3.1.1 是一个补丁版本，主要修复 PDF 文档上传后的数据统计信息不准确问题，以及相关的用户体验改进。

## 问题背景

### 当前问题

1. **PDF 上传后统计信息不准确**
   - 上传 PDF 文件后，`loaded_documents` 统计可能不正确
   - 多页 PDF 的页面信息未正确聚合
   - OCR 图像提取的文档计数混乱
   - 前端显示的 "docs=X, chunks=Y, triplets=Z" 信息不直观

2. **数据一致性问题**
   - `list_indexed_files()` 返回的 chunks 数量与实际索引不一致
   - PDF 页面信息 (`pages` 字段) 未正确收集
   - 重新索引后统计信息未更新

3. **用户体验问题**
   - 上传成功提示信息过于技术化
   - 缺少上传进度的详细反馈
   - PDF 文档的页面数量未显示

## 修复目标

### 核心目标

1. **准确的文档统计** - 确保 `loaded_documents` 准确反映上传的文件数量（而非内部 Document 对象数量）
2. **清晰的页面信息** - 正确统计和显示 PDF 的页面数量
3. **一致的数据展示** - 前后端数据保持一致，避免混淆
4. **改进的用户反馈** - 提供更友好、更直观的上传成功提示

### 非目标

- 不改变核心索引逻辑
- 不修改 API 接口签名（保持向后兼容）
- 不涉及性能优化

## 技术方案

### 1. 后端修复 - 文档统计逻辑

#### 1.1 修复 `ingest_service.py` 的统计逻辑

**文件**: `app/services/ingest_service.py`

**问题分析**:
```python
# 当前代码 (第 36-37 行)
docs = loaders.load_documents(paths=paths)
return {"loaded_documents": len(docs), ...}
```

对于 PDF 文件，`load_documents()` 会返回：
- 每页文本作为一个 Document
- 每个 OCR 图像作为一个 Document
- 导致 `loaded_documents` 数量远大于实际上传的文件数

**修复方案**:
```python
def ingest_paths(
    paths: list[Path],
    reset_vector_store: bool = False,
    metadata_overrides_by_source: dict[str, dict[str, Any]] | None = None,
) -> dict:
    docs = loaders.load_documents(paths=paths)
    if not docs:
        return {"loaded_documents": 0, "chunks_indexed": 0, "triplets_written": 0}

    # 修复：统计实际文件数量，而非 Document 对象数量
    unique_sources = {str((doc.metadata or {}).get("source", "")) for doc in docs}
    unique_sources.discard("")  # 移除空字符串
    files_loaded = len(unique_sources)
    
    # 统计页面信息
    pages_by_source: dict[str, set[int]] = {}
    for doc in docs:
        source = str((doc.metadata or {}).get("source", ""))
        page = (doc.metadata or {}).get("page")
        if source and page is not None:
            pages_by_source.setdefault(source, set()).add(int(page))
    
    if metadata_overrides_by_source:
        for doc in docs:
            source = str((doc.metadata or {}).get("source", "")).strip()
            extra = metadata_overrides_by_source.get(source)
            if extra:
                doc.metadata = {**(doc.metadata or {}), **extra}

    chunks, parent_records = split_documents(docs)
    # ... 其余逻辑保持不变 ...

    return {
        "loaded_documents": files_loaded,  # 修复：返回文件数量
        "chunks_indexed": len(chunks),
        "triplets_written": count_triplets,
        "pages_by_source": {k: len(v) for k, v in pages_by_source.items()},  # 新增：页面统计
    }
```

#### 1.2 修复 `index_manager.py` 的页面聚合逻辑

**文件**: `app/services/index_manager.py`

**问题分析**:
```python
# 当前代码 (第 71-73 行)
page = meta.get("page")
if page is not None:
    entry["pages"].add(str(page))
```

页面被存储为字符串集合，但未正确转换为整数进行排序。

**修复方案**:
```python
# 第 71-73 行修复
page = meta.get("page")
if page is not None:
    try:
        entry["pages"].add(int(page))  # 修复：存储为整数
    except (ValueError, TypeError):
        pass

# 第 121 行修复
for entry in by_file.values():
    entry["pages"] = sorted(entry["pages"])  # 已经是整数，排序正确
    items.append(entry)
```

```typescript
const uploadSummary = [];
uploadSummary.push(`✓ 已上传 ${data.loaded_documents} 个文件`);
if (data.chunks_indexed > 0) {
  uploadSummary.push(`索引了 ${data.chunks_indexed} 个文本块`);

if (data.pages_by_source && Object.keys(data.pages_by_source).length > 0) {

}
if (data.skipped_files && data.skipped_files.length > 0) {
  uploadSummary.push(`跳过: ${data.skipped_files.join(", ")}`);
}
notify(uploadSummary.join(" | "), "success", 4000);
```

```python
import pytest

def test_single_pdf_statistics(tmp_path):

    # 创建测试 PDF（假设有 3 页）
```

## 风险评估

### 低风险

- **向后兼容性**: 所有修改都是向后兼容的，不会破坏现有 API
- **数据迁移**: 不需要数据迁移，现有数据不受影响
