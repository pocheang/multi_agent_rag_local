## 📝 如何使用计划文档

### 版本规划时
1. 查看上一版本的"未来规划"章节
2. 分析当前系统的"之前的状态"
3. 制定本版本的"当前计划"
4. 规划下一版本的"未来规划"

### 开发过程中
1. 跟踪计划执行进度
2. 记录计划变更
3. 更新风险和依赖
4. 调整优先级

### 版本完成后
1. 对比计划与完成报告
2. 分析偏差原因
3. 总结经验教训
4. 改进规划流程

## 🔄 计划与完成报告的关系

### 时间线
```
计划文档 → 开发过程 → 完成报告
(Before)   (Current)   (After)
```

### 内容对应

| 计划文档 | 完成报告 |
|---------|---------|
| 之前的状态 | 执行摘要（背景） |
| 当前计划 | 主要成就（实际） |
| 未来规划 | 后续工作 |
| 预期成果 | 统计数据（实际） |

### 对比分析
通过对比可以：
- 评估计划准确性
- 分析偏差原因
- 改进规划能力
- 优化开发流程
