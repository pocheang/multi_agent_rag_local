## 修复概述

v0.3.1.1 修复了 PDF 文档上传后的数据统计信息不准确问题，并改进了用户体验。

## 核心问题

### 问题描述

上传 PDF 文件后，系统显示的 `loaded_documents` 数量不准确：
- **问题**: 统计的是内部 Document 对象数量（每页文本 + OCR 图像）
- **影响**: 用户看到的数字远大于实际上传的文件数量，造成困惑
- **示例**: 上传 1 个 3 页 PDF，显示 `docs=15`（3 页文本 + 12 个 OCR 图像）

### 用户体验问题

上传成功后的提示信息过于技术化：
```
Indexed: report.pdf | Skipped: none | Visibility: private | docs=15, chunks=45, triplets=120
```

## 修复内容

### 1. 后端修复

#### 1.1 `app/services/ingest_service.py`

**修复前**:
```python
docs = loaders.load_documents(paths=paths)
return {"loaded_documents": len(docs), ...}
```

**修复后**:
```python
docs = loaders.load_documents(paths=paths)

# Count unique source files (not Document objects)
unique_sources = {str((doc.metadata or {}).get("source", "")) for doc in docs}
unique_sources.discard("")
files_loaded = len(unique_sources)

# Collect page information for each source
pages_by_source: dict[str, set[int]] = {}
for doc in docs:
    source = str((doc.metadata or {}).get("source", ""))
    page = (doc.metadata or {}).get("page")
    if source and page is not None:
        try:
            pages_by_source.setdefault(source, set()).add(int(page))
        except (ValueError, TypeError):
            pass

return {
    "loaded_documents": files_loaded,
    "chunks_indexed": len(chunks),
    "triplets_written": count_triplets,
    "pages_by_source": {k: len(v) for k, v in pages_by_source.items()},
}
```

**改进**:
- ✓ `loaded_documents` 现在统计实际文件数量
- ✓ 新增 `pages_by_source` 字段记录每个文件的页面数

## 技术细节

### 数据流

1. **文档加载**: `loaders.load_documents()` 返回多个 Document 对象
2. **统计计算**: 
   - 提取唯一的 `source` 路径作为文件数量
   - 收集每个 source 的页面号集合
3. **返回结果**: 
   - `loaded_documents`: 文件数量
   - `pages_by_source`: 每个文件的页面数
4. **前端展示**: 
   - 计算总页数
   - 构建友好的中文提示

### 向后兼容性

- ✓ 所有修改都是向后兼容的
- ✓ 新增字段使用默认值，不影响现有代码
- ✓ API 接口签名未改变
