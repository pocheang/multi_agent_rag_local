## Guidelines

Use `docs/design/` for:
- Feature design specifications
- Technical architecture proposals
- API design documents
- UX and interaction design artifacts
- Decision records tied to feature design

Historical design specs may remain in their original subdirectories if they are already referenced externally, but new design documents should be placed directly under `docs/design/` or a clearly named subfolder.
